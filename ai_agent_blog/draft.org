- 自宅の PC の OS を Windows から linux(Ubuntu) に置き換えた
- 背景としては、最近 PC でやってることは AI トレーニング、推論、Agent 利用が中心で、
  Windows だとどうしても安定しない、オーバーヘッドが大きいのが気になった
- WSL を利用してもこの傾向は変わらない。むしろオーバーヘッドは余計にかかる
- 自分はゲームをしないので windows でなければ出来ないことは、ほぼないだろうと考えた
- 仮にゲームをすることがあっても、 steam ならそれなりに動きそうだし、
  そもそも Windows を完全に捨てるのではなく、linux と windows マルチブートにするので、
  適宜切り替えれば良い
- そんな訳でメイン OS を linux に置き換えることにした
- なお、メイン OS を linux に置き換えるとはいえ、
  windows はマルチブートで残すので、
  windows からデータにアクセスできるようにする必要がある
- そのためには、どうしても問題になるのがファイルシステム
- wsl を使えば windows からも ext4 等にアクセスは出来るが、とはいえオーバヘッドが大きい
- 一方で linux であれば、幾つかの制限はあるが NTFS にアクセスが出来る
- ということで、基本は既存の NTFS のストレージを残しつつ、
  linux 用のストレージを用意する形で運用を開始した
- なお、現状のストレージはクソ高いので、新規で購入するのではなく、
  既存のストレージを整理し、ドライブ 1 つをフリーにして linux 用とした
- SSD 暴騰直前に 4TB NVMe 買っといてよかった。



- 今回の linux 移行でやったことをまとめておく
  - windows を入れているシステムパーティションを縮小
    - 昔は NTFS のパーティションを縮小するのに linux のツールを使っていて
      結構リスクがあったが、
      今は普通に windows の標準機能にパーティション縮小機能があるため、
      それを使う。
      ただ、windows の標準のパーティション縮小機能は、
      縮小可能なサイズが空きサイズに対して少ない気がする
  - Ubuntu の最新 LTS バージョン(今回は 26.04 )をダウンロードし、
    USB にイメージを書き込み
  - USB ブートから、システムドライブの空けた領域に Ubuntu をインストール
  - 以降は Ubuntu の設定
  - /etc/fstab に NTFS のパーティションを追加
    - このときに ファイルシステムのタイプは ntfs ではなく ntfs3 を指定
    - 最初 ntfs を指定していたが、これだと fuse 経由になってしまいオーバーヘッドが大きくなる
    - ntfs3 にすることで、カーネル直でアクセスできる
    - なお、 ntfs と ntfs3 とで、シンボリックリンクの扱いが異なるため、
      最初から ntfs3 にしておくべき
  - NAS は autofs で nfs で接続
  - ホームにある "ドキュメント" 等のディレクトリ名を英語に変換・固定
  - ~/.cache など、消費量がデカくなるディレクトリを
    別途用意したストレージにシンボリックリンク作成
  - Ubuntu 標準のファイラーがイマイチ自分に合わないので、
    PCMan File Manager を追加
  - ブラウザをインストールし、ブックマークを windows からコピーして復元
  - 日本語入力 IME は、Fcitx5 + SKK を設定
  - emacs をインストールし設定
  - 動画アプリをインストール。今回は VLC を選択
  - Input Remapper をインストールし、キーボード、マウスをカスタマイズ
  - オーディオ再生、画像ビューワは、幾つか試してみたが、
    自分のユースケースにイマイチ合うモノが無かったのと、探すのが面倒になったので、
    Antigravity でバイブコーディグして作成 + メニューに登録
  - ubuntu のショートカットを無効化
    - ウィンドウメニューを開く
  - 起動時、Bluetooth USB アダプタの初期化が上手く動かない事が多かったので、
    usbreset で強制初期化させるツールを Antigravity で作成
  - リモート接続系は remmina を使用
  - リソースモニターは Mission Center をインストール
  - windows の標準ターミナルは設定したキーカスタマイズの一部が反映されなかったので
    konsole を追加でインストール
    
- なお、当初 WSL2 のディスクイメージをマウントして使っていたが、
  何かの拍子にイメージが壊れてしまった。
  実は今回の移行を決める動機にもなった事件でもあるんだが、
  wsl で AI のファインチューニングをしている時に、いつまで経っても終わらず
  仕方がないので強制終了している、
  もしかしたらその時にイメージになんらかのダメージがあり、
  そのダメージが元でディスクイメージが壊れてしまったのかもしれない
  ダメージのあるディスクイメージを Linux でマウントする、
  なんていうのは、ただでさえ互換性に課題があった状況では危険すぎた。
  まぁ、一部のデータはバックアップしてあったので全てのデータを失なった訳ではない。
  とはいえ、あくまでも一部のデータであるので、復旧できないデータも少なくない。
  これは今回の移行で一番の残念ポイント。
  ただ、ディスクイメージがそもそも壊れていたのだとすれば、
  windows で使っていてもいずれは壊れていた可能性は否定できない。
  
  
- 細かい設定は他に諸々あるが、
  上記で Windows 時代と比べてほとんど気にならないレベルで運用できている
- ただ、やはり Linux デスクトップの完成度は Windows と比べると劣ると言わざるをえない。
- 特にマルチメディア系のツールの豊富さは、有料アプリ、無料アプリ含めて windows が圧倒的だろう。
- とはいえ、上記で書いた通り、今は AI エージェントを使うことで、
  コーディングしなくても自分専用のツールを簡単に作れるので、大きな問題では無いかもしれない。
- まぁ、あくまでもちょっとしたツールレベルで、例えばプロが使うような高機能なツールを
  簡単に作れるか？といえば、さすがにない。
- そして一番の問題は、 AI に聞きながら作業すれば多くの作業は可能ではあるが、
  とはいえそれなり以上のリテラシーが必要なのは間違いないので、
  決して万人に進められるものではない。
